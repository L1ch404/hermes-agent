# Jolink Roadmap

> **愿景（Vision）**
>
> 打造一个真正理解软件开发过程的 Agent Runtime。
>
> 长期目标不是做另一个 Agent，而是逐步孵化出一套开放的 Agent Protocol。

---

# 当前阶段（MVP）

状态：🚧 开发中

当前只专注一件事情：

> **让 AI 能像一名有经验的 Java 开发工程师一样排查问题。**

不要急着支持所有语言，不要急着做协议，先把 Java 做到最好。

---

# 第二阶段：自己长期使用（Dogfooding）

目标：

每天开发都使用 Jolink。

重点观察：

* 哪些地方仍然需要自己操作？
* 哪些步骤每天都在重复？
* 哪些信息 AI 无法获取？

原则：

> **重复三次以上的操作，就考虑做成 Tool。**

# 设计理念

## Everything exists to reduce uncertainty for the LLM.

任何设计，都应该帮助 LLM 减少不确定性。

如果 Runtime 可以确定，就不要让 LLM 去猜。

如果 Tool 可以直接回答，就不要让 LLM 推理。

如果开发者每天都会重复一个动作，那么它就值得成为一个 Tool。

---

# 成功标准

Jolink 的成功，不是拥有多少个 Tool。

而是：

**是否真正减少了软件开发过程中的不确定性。**

---

# 开发原则

1. 先自己使用，再开放给别人使用。

2. 不为了抽象而抽象。

3. 不为了协议而设计协议。

4. 从真实开发流程中提炼能力。

5. 每一个 Tool 都应该解决一个真实存在的问题。

6. 理解 → 实践 → 再从实践中理解。

---

> **不要追求做得最多，而要追求做得最有价值。**
